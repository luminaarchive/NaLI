# Do Not Do List

Prohibited actions and designs.

## Absolute Prohibitions

1. **No random unverified packages**: Do not install third-party dependencies from untrusted sources.
2. **No academic cheating terminology**: Avoid using terms like "final-assignment generator", "finished-thesis", or "plagiarism evasion".
3. **No fake features**: Do not mock active payment processing or PDF downloads unless actually implemented and verified.
4. **No public brand naming**: Never expose provider names (OpenAI, Anthropic, etc.) in the public UI. Use NaLI processing language (e.g. Peregrine, Obsidian, Zephyr) instead.
5. **No secret exposure**: Never commit files containing provider API keys, Supabase service role keys, or credentials.
6. **No wildcard root permission requests**: Never ask for root or wildcard permissions via permission tools.
7. **No styling lock breaks**: Do not redesign elements of the public UI without explicit direction.
